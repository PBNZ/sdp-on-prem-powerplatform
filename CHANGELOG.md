# Changelog

All notable changes to sdp-on-prem-powerplatform are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- **Per-action "Learn more" docs** ÔÇö one page per operation (55 across the four connectors)
  under `connectors/<name>/actions/`, each explaining every parameter and showcasing the
  `input_data` formatting variations (JSON envelopes, `search_criteria` grammar, XML
  relationship templates, SQL-in-JSON), grounded in the official on-prem V3 references and
  this repo's live evidence. Generated by `tools/gen_action_docs.py` (ADR-0003), indexed per
  connector at `actions/README.md`.
- `tools/check_action_docs.py` + CI step ÔÇö enforces operation Ôćö doc lockstep: every
  operation carries the Learn-more link, every link resolves, no orphan docs, every
  parameter is mentioned in its page.

- `sdp-service-desk` connector ÔÇö 37 operations across requests (+ notes, worklogs, tasks,
  resolution, close, delete), changes, problems, solutions, and 12 admin-lookup dropdowns.
- `sdp-assets-cmdb-v3` connector ÔÇö 13 operations: assets, CI types, per-type CIs, `_metadata`,
  association relationships (SDP build 15100+).
- `sdp-cmdb-v1` connector ÔÇö 4 operations against the legacy XML CMDB API (builds < 15100).
- `sdp-query` connector ÔÇö Execute Query: one read-only SQL SELECT action.
- `templates/` ÔÇö 8 live-verified Execute-Query SQL templates (build-14990 schema).
- `tools/gen_swagger.py` (curated-table generator for the V3 connectors) and
  `tools/live-test.ps1` (live smoke test + evidence capture).
- WSL2 deploy/redeploy runbook for a disposable SDP 14990 (`docs/deploy-sdp-wsl2.md`).
- Public-tier governance (ADR-0002): Apache-2.0 `LICENSE`, `SECURITY.md`,
  `CODE_OF_CONDUCT.md`, contributor-facing `CONTRIBUTING.md`, issue/PR templates, and a
  definition-validation CI workflow.
- README: prominent vibe-coded-personal-alpha warning (public only so Power Platform can
  import the definitions by URL) plus the four raw import URLs.
- Initial scaffold via RepoKit.

- `tools/check_pp_rules.py` + CI step ÔÇö enforces the Power Platform x-ms rules that
  swagger-cli can't (internal + default Ôçĺ required, internal + required Ôçĺ default).

### Changed

- Connector contact metadata (`info.contact.name`) in all four definitions is now the GitHub
  handle `PBNZ` rather than a personal name, matching the `url` beside it
  ([repo-kit#25](https://github.com/PBNZ/repo-kit/issues/25)). Metadata only ÔÇö no operation,
  path, parameter, or response changed, so definition versions are unchanged and a re-import is
  not required.
- Every operation now carries `externalDocs` (allowed on operations by Microsoft's connector
  schema) **and** a trailing `Learn more: <url>` on its description ÔÇö the About tab's
  "Operation note" shows the description, so the link is visible either way. Definition
  versions bumped: V3 connectors 0.2.0 Ôćĺ 0.3.0, `sdp-cmdb-v1` 0.1.0 Ôćĺ 0.2.0.
- Each connector's description now ends with its own raw import-by-URL, so an imported
  connector self-documents where its definition lives.
- Git history rewritten before publication to scrub a truncated demo-key prefix from an old
  doc example (ADR-0002) ÔÇö all pre-publication commit SHAs changed.

### Fixed

- V3 connectors' `Accept` header is now `required: true` ÔÇö with `required: false` the portal
  wizard refused to save (Swagger Validator rule *PropertyMustBeRequired*: internal parameter
  with a default must be required) and the header would have been silently dropped at runtime.
  This unblocks both file import and import-by-URL for `sdp-service-desk`,
  `sdp-assets-cmdb-v3`, and `sdp-query`.
