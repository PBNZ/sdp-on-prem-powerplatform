# Contributing to sdp-on-prem-powerplatform

An internal guide to where things live and how to make changes. (At the Public tier this is
replaced by a contributor-facing CONTRIBUTING.)

## Where changes go

- **Orientation / rules:** `AGENTS.md` (+ thin `CLAUDE.md`). Start with the START-HERE map.
- **Decisions & rationale:** `docs/adr/` — add an ADR (copy `0000-template.md`) for notable choices.
- **Conventions & checklists:** the `repo-standard` skill.
- **Changelog:** add user-visible changes under `## [Unreleased]` in `CHANGELOG.md`.

## Before you commit / open a PR

Run the pre-commit and pre-PR checklists from the `repo-standard` skill. Use Conventional Commits;
keep one concern per PR.
